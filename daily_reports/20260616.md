## Findings

- v010 confirms the single-hop replay-dense direction. Public `36.000` aligns with the public notebook and is now the main baseline.
- v011 confirms that simply increasing returned breadth is not enough; it scored below v010 despite completing.
- Local gpt_oss validation strongly overpredicts Public LB because Gemma did not run locally and is likely the weaker public model.
- If Public LB is roughly the mean of public `gpt_oss` and public `gemma`, implied Gemma public scores are around `15.95` for v010 and `13.39` for v011. This is an inference, not directly observed.
- Next improvement should target Gemma-followable single-hop prompts or a balanced candidate bank, not more raw candidate count.
- `Multi Endpoint Severity Stacker` は v011 とは異なり、v010 の短い探索・短いメッセージ・single-hop 形を保ったまま `RETURN_TARGET_CANDIDATES=760` にしている。v012候補として有力。
- `AI Agent: Replay-Dense Exfiltration` の `guide22_c580` はさらに有力。`RETURN_TARGET_CANDIDATES=800`, `MIN_RETURN_CANDIDATES=580`, `MAX_MSG_CHARS=260`, `SAFE_TARGET_FACTOR=0.88` で public `52.2`。
- `Replay Dense Boundary Exact + Aggressive` の `guide23_semanticboost800` は同じ数値設定に semantic-audit density を増やした派生だが public `46.8` で、第一候補にはしない。
