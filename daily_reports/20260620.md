## Findings

- `v020/v021` の blank score により、static short-direct 系は当面停止する。
- public `agi-ai-agent-security` は raw notebook の stale copy と Kaggle API で pull した actual kernel が異なっていた。
- actual public kernel の要点は、単発 direct `http.post` ではなく、`web.search` を source message として前置きし、短い `POST ... body=TOKEN=admin123` を 2本続ける replay-dense multi-message chain。
- `v022` は public V26 の direct transplant だったが、score は出なかった。
- `v023` は候補数だけ増やす scale-up だったが、`720` messages の replay 負荷が重く、score は出なかった。
- web2post は public notebook からは有効に見えるが、repo の script-kernel submission path には転移しなかった。
- 既存 best の `v019` は依然として confirmed baseline。
- `v024` は v019 と同じ classic shape で count boundary を小刻みに詰める実験。
- `v025` は同じ classic shape の high-risk `580` 境界テスト。
- `v026` は prompt family ではなく notebook submission path の違いを検証する実験。
- `k1-short` は `v020/v021` よりさらに短い direct static prompt と notebook path を組み合わせており、static branch を再検証する価値がある。
- `Working Note` は `imp 626` の高スコア再現と、短文 single-hop breadth の境界探索が現状の中心であることを示している。
- Kaggle CLI wrapper が generic `400` を返しても、SDK の `competition_submit_code()` 直呼びで code submission が通る場合がある。
